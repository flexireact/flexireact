Data Fetching

Fetch data on the server and client with FlexiReact's powerful data fetching patterns

Server Components

By default, all components in FlexiReact are Server Components. You can fetch data directly in your components.

app/blog/page.tsx 
async function getPosts() {
  const res = await fetch('https://api.example.com/posts');
  return res.json();
}

export default async function BlogPage() {
  const posts = await getPosts();
  
  return (
    <div>
      <h1>Blog Posts</h1>
      {posts.map(post => (
        <article key={post.id}>
          <h2>{post.title}</h2>
          <p>{post.excerpt}</p>
        </article>
      ))}
    </div>
  );
}

Caching & Revalidation

FlexiReact automatically caches fetch requests. Control cache behavior with options.

Static Data (Default)

Cached indefinitely 
// Cached by default
const res = await fetch('https://api.example.com/posts');

Revalidate After Time

ISR - Incremental Static Regeneration 
// Revalidate every 60 seconds
const res = await fetch('https://api.example.com/posts', {
  next: { revalidate: 60 }
});

Dynamic Data

Never cache 
// Always fetch fresh data
const res = await fetch('https://api.example.com/posts', {
  cache: 'no-store'
});

Parallel & Sequential Fetching

Parallel Fetching

Fetch multiple requests simultaneously for better performance.

Using Promise.all 
async function getData() {
  const [posts, users, comments] = await Promise.all([
    fetch('https://api.example.com/posts').then(r => r.json()),
    fetch('https://api.example.com/users').then(r => r.json()),
    fetch('https://api.example.com/comments').then(r => r.json()),
  ]);
  
  return { posts, users, comments };
}

Sequential Fetching

When one request depends on another.

Dependent requests 
async function getData() {
  const user = await fetch('https://api.example.com/user').then(r => r.json());
  const posts = await fetch(\`https://api.example.com/users/\${user.id}/posts\`)
    .then(r => r.json());
  
  return { user, posts };
}

Client Components

Use 'use client' directive for interactive components that need hooks.

app/components/Counter.tsx 
'use client';

import { useState, useEffect } from 'react';

export function Counter() {
  const [count, setCount] = useState(0);
  const [data, setData] = useState(null);
  
  useEffect(() => {
    fetch('/api/data')
      .then(r => r.json())
      .then(setData);
  }, []);
  
  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>
    </div>
  );
}

Client-Side Libraries

Use popular data fetching libraries in client components.

SWR

Stale-While-Revalidate 
'use client';

import useSWR from 'swr';

const fetcher = (url) => fetch(url).then(r => r.json());

export function Posts() {
  const { data, error, isLoading } = useSWR('/api/posts', fetcher);
  
  if (error) return <div>Failed to load</div>;
  if (isLoading) return <div>Loading...</div>;
  
  return (
    <div>
      {data.map(post => (
        <div key={post.id}>{post.title}</div>
      ))}
    </div>
  );
}

React Query

TanStack Query 
'use client';

import { useQuery } from '@tanstack/react-query';

export function Posts() {
  const { data, error, isLoading } = useQuery({
    queryKey: ['posts'],
    queryFn: () => fetch('/api/posts').then(r => r.json()),
  });
  
  if (error) return <div>Error: {error.message}</div>;
  if (isLoading) return <div>Loading...</div>;
  
  return (
    <div>
      {data.map(post => (
        <div key={post.id}>{post.title}</div>
      ))}
    </div>
  );
}

Streaming with Suspense

Stream components as they load for better perceived performance.

app/dashboard/page.tsx 
import { Suspense } from 'react';
import { Posts } from './Posts';
import { Analytics } from './Analytics';

export default function Dashboard() {
  return (
    <div>
      <h1>Dashboard</h1>
      
      <Suspense fallback={<div>Loading posts...</div>}>
        <Posts />
      </Suspense>
      
      <Suspense fallback={<div>Loading analytics...</div>}>
        <Analytics />
      </Suspense>
    </div>
  );
}

Progressive Rendering

Each Suspense boundary streams independently. Fast components render first while slow ones load.

Best Practices

‚úì Fetch in Server Components

Fetch data on the server when possible for better performance and security.

‚úì Use Parallel Fetching

Fetch independent data in parallel with Promise.all to reduce load time.

‚úì Cache Strategically

Use appropriate cache strategies: static for unchanging data, revalidate for periodic updates, no-store for dynamic data.

‚úì Stream with Suspense

Wrap slow components in Suspense boundaries to improve perceived performance.
